This is the API reference for integrating with Grid Products. It provides detailed information on endpoints, authentication, and examples for retrieving, updating, and managing your product data.

## Overview

The Grid Products API is a RESTful API that allows you to manage your product catalog programmatically. You can retrieve, create, update, and delete products and product types across multiple spaces and environments.

### Key Features

- **Multi-region support**: Available in EU, US, IN, AU, and UAE regions
- **Multi-language support**: Handle product information in multiple languages
- **Space-based organization**: Manage products across different spaces/locations
- **Batch operations**: Process up to 100 records per batch for efficiency
- **Flexible querying**: Advanced filtering, searching, and pagination
- **Variant management**: Handle product variants with pricing and inventory

**Configuration Notes:**

1. **URL Placeholders**: Replace `<tenant-id>`, `<environment>`, and `<space-id>` with your actual values.
2. **Data Residency**: Replace `<data-residency>` with one of the following: `eu`, `us`, `in`, `au`, or `uae`.
3. **Headers**: Set the `x-api-key` or `Authorization` header to your generated access token from the Grid Console.

### Authentication

To use the Grid Products API, every request requires an **access token**. Here's how to generate one:

1. Go to the **<SquaresFour weight="fill" />Apps &rarr; <Terminal weight="fill" />Developer** tab in the Phygrid Console.
2. Generate an access token.
3. Add the token as an `x-api-key` in your request headers.

  <ApiSection type="left">
    ### POST Push Products

    This API allows you to upload or merge products following the `GridProduct` format. The only required field is `productGroupId`, while other fields are optional.

    #### Body

    The body of the request should be an Array of [GridProducts](#) JSON format using the `content-type` header `application/json`.

    <ParameterList title="Parameters">
      <Parameter name="data" type="Array<Partial<GridProduct>>" required>
        An array of Partial GridProduct. This can be a list of products in full GridProduct format or Partial GridProduct to update specific product fields.
      </Parameter>
    </ParameterList>

    #### Payload Behavior

    Please ensure that the `productGroupId` is always included in the payload. You have the flexibility to send either a single product field, multiple fields, or the entire product in your payload.

    **Key behaviors:**
    - **Replace operations**: Fields like `productName`, `productDescription`, `productType` will be completely replaced
    - **Merge operations**: Arrays like `variants`, `productPriceList`, `productStatus` will be merged based on key fields
    - **Limitations**: 100 products per batch

    Reference: [GridProduct](#)

  </ApiSection>

  <ApiSection type="right">

    ### PATCH (Update) Variants

    Updates variants with the fields specified in the data object without requiring the product's `productGroupId`. Shallow-update is performed. Fields not passed in this call stay the same. Fields containing Arrays or Objects (`productPriceList` and `productItemQuantity`) with the matching specified variant's `productId` and `spaceId` only will be upserted.

    <ParameterList title="Parameters">
      <Parameter name="data" type="Array<Partial<VariantUpdateFields>>" required>
        List of variants level information with fields in VariantUpdateFields format to update to the Database
      </Parameter>

      <Parameter name="upsertSpaceIds" type="boolean">
        Flag to upsert spaceId values from the productItemQuantity and productPriceList fields to the main product's spaceIds field. Example: `true`
      </Parameter>
    </ParameterList>

    Reference: `[VariantUpdateFields](data-model)`

  </ApiSection>

  <ApiSection type="right">
    <ApiEndpoint method="PATCH" endpoint="{base-url}/{tenant-id}/{environment}/variants">
      
      ```bash
      curl -X PATCH \
        '{base-url}/{tenant-id}/{environment}/variants' \
        -H 'Content-Type: application/json' \
        -H 'x-api-key: your-access-token-here' \
        -d '{
          "data": [
            {
              "productId": "100001-001",
              "productPriceList": [
                {
                  "listPrice": 39.99,
                  "isoCurrencyCode": "USD",
                  "spaceId": "space-001"
                }
              ],
              "productItemQuantity": [
                {
                  "productItemQuantity": 75,
                  "spaceId": "space-001"
                }
              ]
      ```
