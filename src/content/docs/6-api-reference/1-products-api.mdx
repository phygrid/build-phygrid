This is the API reference for integrating with Grid Products. It provides detailed information on endpoints, authentication, and examples for retrieving, updating, and managing your product data.

## Overview

The Grid Products API is a RESTful API that allows you to manage your product catalog programmatically. You can retrieve, create, update, and delete products and product types across multiple spaces and environments.

### Key Features

- **Multi-region support**: Available in EU, US, IN, AU, and UAE regions
- **Multi-language support**: Handle product information in multiple languages
- **Space-based organization**: Manage products across different spaces/locations
- **Batch operations**: Process up to 100 records per batch for efficiency
- **Flexible querying**: Advanced filtering, searching, and pagination
- **Variant management**: Handle product variants with pricing and inventory

**Configuration Notes:**

1. **URL Placeholders**: Replace `<tenant-id>`, `<environment>`, and `<space-id>` with your actual values.
2. **Data Residency**: Replace `<data-residency>` with one of the following: `eu`, `us`, `in`, `au`, or `uae`.
3. **Headers**: Set the `x-api-key` or `Authorization` header to your generated access token from the Grid Console.

### Authentication

To use the Grid Products API, every request requires an **access token**. Here's how to generate one:

1. Go to the **Apps** &rarr; **Developer** tab in the Phygrid Console.
2. Go to Access Token tab and generate an access token.
3. Add the token as an `x-api-key` in your request headers.

    ### Create Products
    <AuthBadge>ðŸ”’ AUTH REQUIRED</AuthBadge>

    This API allows you to upload or merge products following the `GridProduct` format. The only required field is `productGroupId`, while other fields are optional.

    #### Body

    The body of the request should be an Array of [GridProducts](#) JSON format using the `content-type` header `application/json`.

    <ParameterList title="Parameters">
      <Parameter name="data" type="Array<Partial<GridProduct>>" required>
        An array of Partial GridProduct. This can be a list of products in full GridProduct format or Partial GridProduct to update specific product fields.
      </Parameter>
    </ParameterList>

    #### Payload Behavior

    Please ensure that the `productGroupId` is always included in the payload. You have the flexibility to send either a single product field, multiple fields, or the entire product in your payload.

    **Field Update Behavior:**

    **Replace Operations:** Fields like `productName`, `productDescription`, `productType`
     will be completely replaced with new values.
    - `productName` - The product name will be updated with the new value
    - `productShortDescription` - Product short description will be updated
    with the new value
    - `productInternalName` - Product internal name will be updated with
    the new value
    - `storageInstructions` - Product storage instructions will be updated
    with the new value
    - `consumerStorageInstruction` - Product consumer storage instructions
     will be updated with the new value
    - `productShippingInstruction` - Product shipping instructions will be
    updated with the new value
    - `productType` - Array of product type UUIDs will be updated with the
    new value
    - `productLabel` - Product label will be updated with the new value
    - `productTags` - Product tags will be updated with the new value
    - `brand` - Product brand will be updated with the new value
    - `productVendor` - Product vendor will be updated with the new value
    - `productDescription` - Product description will be replaced with the
     new value
    - `spaceIds` - Array of space IDs where the product is available

    **Merge Operations:** Arrays will be merged based on key fields:
    - `variants` - Will merge and update product variants array based on
     productId and productGroupId
    - `productFeature` - Will merge and update product features array based
     on productId
    - `productPriceList` - Will merge and update product prices array based
     on productId and spaceId
    - `productStatus` - Will merge and update product status array based
    on spaceId
    - `catalogPageLocationProduct` - Will merge and update product catalogue
     array based on productId and productGroupId
    - `customProperties` - Will merge and update product custom properties
    array based on key and spaceId
    - `productItemQuantity` - Will merge and update product quantity array
    based on productId and spaceId

    **Note:** The `spaceIds` array can be included in the request payload
    and will also be automatically populated based on the values of
    `productPriceList`, `productStatus`, and `productItemQuantity`.

    **Limitations:** 100 products per batch

    Reference: [GridProduct](#)

  </ApiSection>
